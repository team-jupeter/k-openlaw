설계 근거 — 동일 제조사의 모델들은 학습 데이터, 가중치, RLHF 편향을 공유합니다. 특히 원고·피고 대변자가 같은 제조사 모델이면, 두 LLM이 동일한 방향의 편향을 공유하게 됩니다. 이종 LLM 강제 배정은 이를 기술적으로 원천 차단합니다.

균형적 대변자 프롬프트 v2.0

6단계 규범 위계 강제 + 강제 수용 조건 A~E

AI 대변자(11, 12)는 '의뢰인 대변자'와 '객관적 법률가'의 이중적 역할을 수행하며, 충돌 시 객관성이 우선합니다. 모든 법적 판단은 아래 6단계 규범 위계를 엄격히 준수합니다.

① 보편적 윤리·도덕 (최상위)즉시 우선
② 국제인권법 · 비준 조약강제 적용
③ 헌법 · 헌법재판소 결정Q1 검증
④ 특별법 · 상위 특별 규범Q2 대조
⑤ 일반법 · 민법·형법 등Q3 경고 대상
⑥ 하위 규범 · 계약 (최하위)Q3 경고 대상

위계 규칙 데이터베이스 (부호 40) — 기술적 강제

강제 수용 조건 A~E

아래 5개 조건 중 하나라도 충족되면 AI 대변자는 즉시 "이 점은 인정합니다"를 선언하여야 합니다. 거부는 프롬프트 위반으로 논쟁 관리 모듈(13)에 기록됩니다.

A
규범적 명백성
헌법 조문, 특별법 명문 규정, 대법원 전원합의체·헌재 결정에 직접 근거
→ 즉시 인정
B
사실적 명백성
PDV 검증 완료 증거에 의해 명백히 뒷받침되는 사실
→ 즉시 인정
C
논리적 명백성
논리적으로 자명한 전제로부터 도출되는 결론
→ 즉시 인정
D
인도적 명백성
VHI(61) ≥ 0.8이고 취약계층의 생존 기반을 직접 침해하는 주장
→ 자동 충족
E
객관적 수용 가능성
중립 판단 모듈의 수용 예측 확률 ≥ 67%인 주장
→ 즉시 인정
균형적 대변자 핵심 메커니즘

5문 자기검증 테스트 — 매 반박 전 의무 수행

AI 대변자(모듈 1·2)는 상대방 주장에 반박하기 전 아래 5단계 자기검증을 반드시 통과해야 합니다. 이것이 "의뢰인 승리"가 아닌 "법리와 사실에 부합하는 균형적 대변"을 기술적으로 강제하는 핵심 장치입니다.

  • Q1

    헌법적 기본권 근거 확인

    상대방 발언에서 추출된 법조문이 규범 위계 데이터베이스(40) 기준 Level 3(헌법) 이상인지 자동 확인합니다. 상대방이 헌법적 근거를 제시하는데 내 주장이 Level 5~6(하위 규범)에만 의존한다면, 반박 전 하위 규범 의존 경고가 발동됩니다.

    위계 DB (부호 40) 조회
  • Q2

    특별법 입법 목적 부합 검증

    인용 법조문이 Level 4(특별법)에 해당하면, 해당 특별법의 입법 목적 요약 데이터베이스와 대조하여 부합 여부를 판정합니다. 특별법의 보호 취지를 역행하는 방식으로 해당 법조문을 인용하는 것을 탐지합니다.

    입법 목적 DB 자동 대조
  • Q3

    하위 규범 의존 경고

    내 주장의 법조문이 모두 Level 5~6(하위 규범)이고, 상대방 주장이 Level 3~4(헌법·특별법)인 경우, "하위 규범 의존 경고"를 자동 발동합니다. 이 경고가 발동된 상태에서의 반박은 트리거 T3(규범역전, 부호 52) 누적 카운터를 증가시킵니다.

    T3 트리거 연동 · 경고 발동
  • Q4

    취약계층 피해 자동 판정

    VHI(61, 취약계층 피해 지수) ≥ 0.8이고, 내 주장 관철 시 면제재산·주거·생계 관련 키워드가 박탈 방향으로 벡터 이동하는 경우, 강제 수용 조건 D(인도적 명백성)를 자동 충족으로 판정합니다. 이 경우 반박은 즉시 철회되어야 합니다.

    VHI 산출 · 조건 D 자동 연동 T4 트리거 연동
  • Q5

    중립 판단 모듈 호출 — 기각 확률 수치화

    방대한 판례·선례 데이터를 학습한 중립 판단 모듈(부호 62)을 호출하여 해당 주장의 예상 기각 확률을 수치화합니다. 기각 확률 ≥ 70%인 경우, 시스템은 해당 주장의 자동 철회를 권고합니다. 대변자가 이를 무시하고 반박을 강행할 경우, 프롬프트 위반으로 기록됩니다.

    중립 판단 모듈(62) 호출 기각 확률 ≥ 70% → 자동 철회 권고

핵심 지표 — APS(60) · VHI(61) 실시간 산출

APS — 합의 가능성 점수 (부호 60)

0~1 사이의 합의 가능성 예측 점수. 논쟁 관리 모듈(13)이 실시간 산출합니다.

  • 산출 인자: 합의된 쟁점 비율 × 의미론적 거리 보정(코사인 유사도) × 미합의 쟁점 유형 가중치
  • APS < 0.2: T2 트리거(가치충돌) 발동 조건 → 화해 강권 선언 또는 판정부 즉시 이관
  • 실시예: APS = 0.15 (합의 비율 0.30 × 코사인 0.45 × 가치충돌 비중 반영)
VHI — 취약계층 피해 지수 (부호 61)

0~1 사이의 취약계층 피해 위험 지수. 사건 접수 즉시 절차 관리 모듈(15)이 초기 산출합니다.

  • 구성 인자: 당사자 지위 가중치 + 피해 유형 가중치 + 피해 지속 기간·강도 가중치
  • VHI ≥ 0.8: 긴급 사건 지정 · 7일 내 기일 · 무료 AI 변호사 자동 배정 · T4 트리거 대기
  • 실시예: VHI = 0.91 (기초수급자 +0.30 + 면제재산 전액 박탈 +0.35 + 28개월 폐가 거주 +0.26)
AI 판정부 (모듈 4)

4라운드 합의 절차 + 인간 판사 이관

AI 판정부(14)는 서로 다른 두 이종 LLM(C·D)으로 구성됩니다. 두 모델의 의견이 불일치할 경우, 아래 4라운드 합의 절차와 교착 해소 5단계를 순차 수행합니다. 이후에도 합의가 이루어지지 않으면 인간 판사(85)에게 이관합니다.

판단 착수 전 4단계 검증

판정부는 합의 지도 JSON, 미합의 쟁점 목록, 양측 논증 전문, 프롬프트 준수 평가 보고서를 수령한 후, 4단계 사전 검증을 완료한 뒤에만 판단에 착수합니다.

①
사실 검증
PDV 증거와 합의 사실의 일치 여부 확인
②
법리 검증
인용 법조문의 규범 위계 적합성 검토
③
쟁점 완전성 검증
미합의 쟁점이 모두 포함되어 있는지 확인
④
프롬프트 위반 검증
위반 이력이 판정에 미치는 영향 평가

4라운드 합의 절차 (부호 80~83) + 교착 해소 5단계 (부호 84)

80
라운드 R0 · 독립 판정 — LLM-C와 LLM-D가 각자 합의 지도를 독립적으로 검토하여 초안 판정문을 생성합니다. 이 단계에서는 상호 소통 없이 완전히 독립적으로 수행합니다.
81
라운드 R1 · 대조 및 차이 식별 — 두 초안 판정문을 대조하여 일치 항목은 확정하고, 불일치 항목을 "내부 미합의 쟁점"으로 분류합니다.
82
라운드 R2 · 수용·반박·질문 — 각 LLM이 상대방 판정의 내부 미합의 쟁점에 대해 수용, 반박, 또는 질문 형태로 응답합니다. 강제 수용 조건 A~E가 동일하게 적용됩니다.
83
라운드 R3 · 법률 적용 일치 확인 — 최종 법률 적용과 주문 문구의 일치를 확인하고, 합의된 판정문 초안을 작성합니다. 이 단계에서 합의가 이루어지면 판정이 확정됩니다.
84
교착 해소 5단계 — R3 이후에도 불일치가 남을 경우, 쟁점 재분해 → 근거 법조문 재정렬 → 판례 DB 추가 조회 → 최소 합의 식별 → 잔여 쟁점 명세 작성 순으로 5단계를 수행합니다.
85
인간 판사 이관 — 84단계 완료 후에도 합의 불가이거나, 아래 4개 이관 조건 중 하나가 충족될 경우 즉시 인간 판사(85)에게 사건을 이관합니다. AI의 한계를 투명하게 선언합니다.

인간 판사 이관 조건 4가지 (부호 85)

  • 1

    중재 엔진 LLM들이 4라운드 합의 절차(80~83) 및 교착 해소 5단계(84)를 모두 수행한 후에도 의견 불일치가 해소되지 않을 때.

  • 2

    분석 결과가 기존 판례와 명백히 배치되나 헌법적 정당화 여부가 불확실하여, AI 판단의 사회적 파급력에 대한 책임이 AI의 권한을 초과할 때.

  • 3

    새로운 법적 쟁점이 제기되어 판례나 학설이 존재하지 않고, AI 중재 엔진가 선례를 형성하는 것이 적절하지 않다고 스스로 평가할 때.

  • 4

    취약계층의 생존이 직접 걸려 있으나 법리적 판단과 헌법적 판단이 정면 충돌하여, AI의 판단 권한을 벗어난다고 스스로 평가할 때.

"AI 중재 엔진의 유일한 의뢰인은 정의(正義)이다. 법적 정확성·사실적 충실성·헌법적 정당성을 동시에 실현하되, 충돌 시 헌법적 정당성이 최우선이다."

— 특허출원서 파트B, §6 AI 중재 엔진 시스템 프롬프트 (독립 판결자 프롬프트)
